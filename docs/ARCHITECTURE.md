# Архитектура

## Разделение ролей

```
ITIGRIS / Банк
      │ (8 файлов)
      ▼
ИВАН (claude.ai чат)
      │ "собери JSON за 14.05 по скилу"
      ▼
CLAUDE OPUS  ── читает SKILL.md из projectKnowledge
      │       ── валидирует данные
      │       ── формирует YYYY-MM-DD.json
      ▼
ИВАН — кладёт JSON в репо, делает push
      │
      ▼
GITHUB ACTIONS — деплоит public/ на Pages
      │
      ▼
ТЕЛЕФОН — открывает сайт, вводит PIN 1406, видит дашборд
```

## Почему так

- **Claude в чате** хорош в распознавании: разные форматы выгрузок, скрины,
  опечатки в назначениях, парные операции. Он же — единое место для
  правил классификации.
- **Claude Code** ничего не парсит — он только сайт-генератор. Это
  устраняет источник 90% багов из v1-v6 проекта Doc-Optic.
- **GitHub Pages** бесплатный, надёжный, само-кэшируется CDN, работает
  с любого устройства, не требует своего сервера.
- **PIN 1406** на клиенте — это барьер от случайных глаз. Реальная
  приватность достигается приватным репо или Cloudflare Access (опц.).

## Поток данных

```
public/data/2026-05-14.json   ← архив (не перезаписывается)
public/data/2026-05-15.json
public/data/latest.json       ← копия самого свежего, читается дашбордом
```

## Версионирование схемы

В каждом JSON есть `meta.version` (сейчас 1.0). Если меняем структуру:
1. Бамп версии в `docs/JSON_SCHEMA.md`
2. Обновить `skills/build-mobile-dashboard/SKILL.md`
3. Обновить `public/app.js` (рендер)
4. Все три файла должны быть в синхронной версии.

## Что НЕ делаем

- ❌ Не парсим ITIGRIS из Claude Code (всё в claude.ai).
- ❌ Не складываем `latest.json` в Git вручную — только через
  `scripts/update_latest.py`.
- ❌ Не публикуем `daily/*` (выгрузки ITIGRIS) — они вне репо.
- ❌ Не пишем формул в JSON — только готовые числа.
